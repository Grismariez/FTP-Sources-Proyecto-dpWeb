Nuestra página web fue creada bajo la plataforma: W.A.M.P.



Una plataforma web es un conjunto de sistemas o herramientas que sirven para crear o para poner a funcionar sitios o aplicaciones web. La eleccion de una u otra plataforma dependera de factores como: precio, velocidad, flexibilidad, soporte, entre otros.
Para ser considerada una plataforma web como mínimo debe reunir los siguientes elementos:
  • Sistema Operativo
  • Servidor Web
  • Gestor de Base de Datos
  • Lenguaje de Desarrollo

TIPOS DE PLATAFORMAS WEB:


LAMP (Linux, Apache, MySQL, PHP):

Se conoce con el término LAMP a la plataforma para desarrollo y ejecución de aplicaciones web de alta perfomance dada por el uso en conjunto de Linux, Apache, MySQL y PHP. El nombre LAMP se debe justamente a las iníciales de estas 4 tecnologías. LAMP está considerada como una de las mejores herramientas para desarrollar y ejecutar aplicaciones web, sus tecnologías componentes por separado son de altísima calidad y en conjunto sus virtudes se multiplican.

A diferencia de gran cantidad de otras tecnologías de uso similar, LAMP no tiene publicidad propia, no hay una corporación detrás de su desarrollo, sino una comunidad de entusiastas y programadores de primer nivel que trabajan con la calidad del producto como premisa. Tanto Linux como Apache, MySQL y PHP son Software de código abierto por lo que si usted escucha hablar de LAMP será verdaderamente por sus virtudes, no por publicidad.
    Ventajas:
    • Disponibilidad y bajos costos
    • Velocidad y mejor tiempo de actividad
    • Mejor accesibilidad y ambiente más amigable para los buscadores
    • Mejor plataforma para la implementación y manejo de reescritura de URL
    • Permite actualización de código para nuevas funcionalidades del servidor
    • Servidor de BD MySQL más rápido para consultas de lectura y sitios no demasiado grandes
    Desventajas:
    • Rendimiento bajo de MySQL para sitios muy grandes
    • Dificulta el trabajo para los que están acostumbrados a Windows

WAMP (Windows, Apache, MySQL, PHP):

Es el acrónimo usado para describir un sistema de infraestructura de internet que usa las siguientes herramientas:
  • Windows, como S.O.
  • Apache, como servidor web
  • MySQL, como gestor de bases de datos
  • PHP (generalmente), Perl o Python, como lenguajes de programación.


       


El uso de una plataforma como WAMP permite servir páginas html a internet, además de poder gestionar datos en ellas, al mismo tiempo un WAMP, proporciona lenguajes de programación para desarrollar aplicaciones web. Existe el software llamado WAMP5 que facilita su manejo.
    Ventajas:
    • Fácil instalación con el software WAMP5
    • Ideal para utilizar como servidor de desarrollo
    Desventajas:
    • Dificultad a la hora de instalar los subsistemas por separado.

LAMDA (Linux, Apache, [Mono|Maverick.NET], DB4O, ActiveRecord):
    Ventajas:
    • Mayor robustez, es decir multiplataforma
    • Mayor flexibilidad al soportar múltiples BD
    • Mismo funcionamiento del framework Maverick.NET con Mono que con el framework Microsoft.NET
    Desventajas:
    • Presenta algunos fallos al momento de instalarse y ejecutarse

XAMPP (X=Cualquier S.O, Apache, MySQL, PHP, [Perl|Python]):

Es un servidor de plataforma libre, es un software que integra en una sola aplicación, un servidor web Apache, interpretes de lenguaje de scripts PHP, un servidor de base de datos MySQL, un servidor de FTP FileZilla, el popular administrador de base de datos escrito en PHP, MySQL, entre otros módulos. Su uso es gratuito.

    Ventajas:
    • Xampp es una herramienta muy práctica que permite instalar el entorno MySQL, Apache y PHP, suficiente para empezar proyectos web o revisar alguna aplicación localmente. Además trae otros servicios como servidor de correos y servidor FTP
    • Fácil instalación de apache y php sobre Unix, sobre todo si dispone de un manejador de paquetes
    • Fácil instalación y configuración mínima
    Desventajas:
    • Soporte escaso de MySQL desde consola
    • Dificultad para configurar aplicaciones de terceros
    • Actualización individual de versiones deshabilitada
    • El instalador de Xampp obliga a reinstalar todos los ficheros al publicarse una actualización del sistema

WXCF (Windows, X= [Apache|IIS], ColdFusion):

Coldfusion es un servidor de páginas, una herramienta y un lenguage de programación que permite programar páginas web fácilmente a través de etiquetas similares a HTML y que puede instalarse además de Windows en Mac OS X, Linux y Solaris. Actualmente pertenece a Adobe y anteriormente a Macromedia. Permite manejar bases de datos como Sybase, Oracle, MySQL, SQL Server, o Access mediante el standard SQL.
    Ventajas:
    • Integra varias tecnologías, entre ellas Java
    • Fácil programación a través de etiquetas o tags
    • Amplias opciones para la configuración del servidor y en modo gráfico
    • Mejores funciones para leer y escribir en HDD del servidor basadas en tags
    • La instalación de Coldfusion permite elegir sobre que servidor web va a correr (Apache o IIS)
    Desventajas:
    • Ofertas escasas de hosting para esta plataforma
    • Servidor muchas veces inestable a pesar de su robustez

XATMJ (X=Cualquier S.O, Apache, Tomcat, MySQL, JSP):

Este tipo de plataforma web tampoco es muy común encontrarla en los proveedores de hosting más populares. Tomcat es un servidor web con soporte de servlets y JSPs de Sun Microsystem y que normalmente es combinado con el servidor web Apache aunque puede funcionar por sí sólo.

Si vas a realizar páginas web en JSP ésta es la mejor opción. También podrías utilizar WebSphere Application Server (WSAS) en lugar de Tomcat como servidor, pero ten presente que: este otro sotfware es de pago y Tomcat es gratuito.
    Ventajas:
    • Tomcat es muy fiable a la hora de trabajar con ésta herramienta
    • Funciona en cualquier sistema operativo que disponga de la máquina virtual Java
    Desventajas:
    • Tomcat no dispone de un entorno integrado con una sofisticada GUI
    • Tomcat sólo puede trabajar con un único servidor; no puede utilizarse en clústeres

WIMA (Windows, IIS, MS SQL Server, ASP.NET):

Este sistema junto a LAMP es de los más utilizados para desarrollar aplicaciones web basadas en ASP clásico y ASP.NET. También se puede instalar PHP o Perl bajo IIS, y es la forma en la que los proveedores de hosting ofrecen ASP y PHP en el mismo servidor.

    Ventajas:
    • La mejor opción para crear aplicaciones web en la actualidad
    • Mejor opción existencial al contar con la tecnología de ASP.NET
    • MS SQL Server tiene un potente entorno gráfico de administración
    • MS SQL Server ofrece a múltiples usuarios grandes cantidades de datos simultáneamente
    Desventajas:
    • Vulnerabilidades encontradas en el sistema en general
    • El servidor MS SQL Server tampoco es multiplataforma
    • El servidor IIS no es multiplataforma, sólo funciona bajo Windows
    • Limitaciones en IIS en las versiones que no son de la familia “Server”

APPSERV:

Es una herramienta OpenSource para Windows con Apache, PHP, MySQL, PHPMyAdmin.

    Ventajas:
    • Licencia GNU, es decir, es un sistema gratuito
    • Muy eficaz para desarrollar y depurar scripts PHP
    • Su versión 2.4 es muy estable, así como las demás que le suceden a ésta
    • Su versión 2.5 provee el más nuevo y experimental paquete de subsistemas
    • Puede ejecutarse en el equipo local para un número ilimitado de sitios web
